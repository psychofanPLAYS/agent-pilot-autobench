# Example: Flag-Ladder Dry Run

This sanitized artifact was generated by the real CLI with:

```powershell
agent-autobench autoresearch --model "G:\models\showcase-model.gguf" --flag-ladder --dry-run
```

Dry-run mode writes the launch plan without starting `llama-server`. The commands below
are evidence of orchestration behavior, not benchmark results or recommended settings.

| Profile | Hypothesis | Distinguishing arguments |
| --- | --- | --- |
| `L0-baseline` | Establish a plain localhost baseline. | `--parallel 1 --flash-attn on --cont-batching` |
| `L1-parallel` | Measure the cost or benefit of parallel slots. | `--parallel 4` |
| `L2-kv-unified` | Compare unified KV behavior with the baseline. | `--parallel 4 --kv-unified` |
| `L3-ram-cache` | Measure prompt RAM-cache overhead. | `--kv-unified --cache-ram 22528` |
| `L4-cache-reuse` | Measure cache-reuse overhead. | `--kv-unified --cache-reuse 512` |
| `L5-checkpoints` | Measure context-checkpoint overhead. | `--kv-unified --ctx-checkpoints 16` |
| `L6-q8-kv` | Measure the q8 KV quality, speed, and memory tradeoff. | `--cache-type-k q8_0 --cache-type-v q8_0` |
| `T12-threads` | Start the thread sweep from the q8 KV profile. | `--threads 12 --threads-batch 12` |
| `T16-threads` | Continue the same controlled thread sweep. | `--threads 16 --threads-batch 16` |
| `T24-threads` | Continue the same controlled thread sweep. | `--threads 24 --threads-batch 24` |
| `T32-threads` | Complete the same controlled thread sweep. | `--threads 32 --threads-batch 32` |

Every profile also receives the benchmark-managed model path, `127.0.0.1` host, dedicated
port, context size, batch sizes, GPU-layer count, metrics, slot, and no-WebUI settings.
User-supplied extra arguments cannot override the managed model, host, or port.

The live run records a transcript, summary, bounded server logs, warnings, accuracy, TTFT,
throughput, and exact command for each profile before selecting a result.
